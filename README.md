## Описание

Скрипт обращается к REST API InSales, получает список товаров с вариантами, изображениями и категориями, и конвертирует их в формат CSV, совместимый со встроенным импортом WooCommerce.

**Что переносится:**
- Названия и описания товаров (HTML)
- Цены и остатки
- Варианты (модификации): размеры, цвета и т.д.
- Категории (с иерархией)
- Изображения (по URL)
- SKU (артикулы)

**Что НЕ переносится (требует ручной настройки):**
- Заказы и клиенты
- Скидки и промокоды
- Настройки доставки и оплаты
- SEO-мета-поля (если не настроены в InSales)

## Возможности

- Фильтрация: выгружаются только товары, находящиеся в продаже (в наличии и доступные)
- Обработка пакетами по 10 товаров с промежуточной записью
- Автоматическая обработка лимитов API InSales (429 Retry-After)
- Пагинация: поддержка магазинов с любым количеством товаров
- Иерархия категорий: категории переносятся с вложенностью
- Варианты товаров: variable/simple продукты определяются автоматически

## Настройка

### 1. Получение API-ключа InSales

1. Войдите в админку вашего магазина InSales
2. Перейдите: **Расширения → Разработчикам**
3. Нажмите **Создать ключ доступа**
4. Скопируйте:
   - **Идентификатор** (длинная строка из букв и цифр)
   - **Пароль** (длинная строка из букв и цифр)

> **Важно:** ключ даёт полный доступ к данным магазина. Не передавайте его третьим лицам.

### 2. Создание конфигурации

```bash
cp config.example.py config.py
```

### 3. Заполнение конфигурации

Откройте `config.py` и вставьте данные:

```python
# Домен магазина — из URL вашей админки
# Если URL: https://myshop-bqr991.myinsales.ru/admin
# то SHOP = "myshop-bqr991.myinsales.ru"
SHOP = "myshop-bqr991.myinsales.ru"

# Идентификатор и пароль API — из панели Разработчикам
API_ID = "79f3d62dcddec08763ce2e4b1702f2c4"
API_PASSWORD = "72236c31baba7f835dc0ee51c343a794"

# Количество записей на страницу (макс. 250)
PER_PAGE = 250
```

> **Не пушьте `config.py` в публичный репозиторий** — он содержит секретные ключи.

## Импорт в WooCommerce

1. Откройте админку WordPress
2. Перейдите: **Products → All Products**
3. Нажмите **Import**
4. Выберите файл `output/products.csv`
5. Нажмите **Continue**
6. Проверьте маппинг колонок (обычно определяется автоматически)
7. Нажмите **Run the Importer**
8. Дождитесь завершения

> **Совет:** перед полным импортом протестируйте на одном пакете (`batch_001.csv`).

## Как это работает

### Этапы миграции

```
1. Загрузка категорий
   InSales API → categories.json → дерево категорий

2. Загрузка списка товаров
   InSales API → products.json → фильтр (available + quantity > 0)

3. Обработка пакетами по 10 товаров:
   Для каждого товара:
     - Загрузка полных данных (product/{id}.json)
     - Загрузка вариантов (product/{id}/variants.json)
     - Загрузка изображений (product/{id}/images.json)
   → Формирование CSV строк
   → Запись batch_NNN.csv

4. Сборка итогового файла
   Все batch_*.csv → products.csv
```

### Маппинг полей

| InSales | WooCommerce CSV |
|---|---|
| `title` | `Name` |
| `description` | `Description` |
| `sku` | `SKU` |
| `price` | `Regular price` |
| `quantity` | `Stock` |
| `category_id` | `Categories` (через маппинг) |
| `variants[].title` | `Attribute N name/value(s)` |
| `images[].url` | `Images` |

### Фильтрация «в продаже»

Товар считается находящимся в продаже, если:
- Есть хотя бы один вариант с `available = true` И `quantity > 0`
- ИЛИ (нет вариантов) и сам товар с `quantity > 0`

## Решение проблем

### Ошибка «Создайте config.py»

Убедитесь, что вы скопировали шаблон:
```bash
cp config.example.py config.py
```

### Ошибка 401 Unauthorized

Проверьте API_ID и API_PASSWORD в `config.py`. Ключ создаётся в: **Расширения → Разработчикам**.

### Ошибка 429 Too Many Requests

Скрипт автоматически обрабатывает лимиты API (500 запросов / 5 минут). Если ошибка повторяется — уменьшите `PER_PAGE` в `config.py`.

### Таймаут при запуске

При большом каталоге (1000+ товаров) скрипт может работать долго из-за поштовой загрузки данных по каждому товару. Это нормально — каждый товар требует 3 API-запроса (данные + варианты + изображения).

### CSV не импортируется в WooCommerce

- Убедитесь, что файл в кодировке UTF-8
- Проверьте, что колонки соответствуют схеме WooCommerce
- Попробуйте импортировать один пакет (`batch_001.csv`) для теста

### Изображения не загружаются

WooCommerce поддерживает импорт изображений по URL. Убедитесь, что:
- URL изображений доступны из интернета
- Изображения не защищены авторизацией
- URL начинаются с `https://`

## FAQ

### Сколько времени занимает миграция?

Зависит от количества товаров. На каждый товар уходит 3 API-запроса + задержка между пакетами. Примерно:
- 100 товаров → 5–10 минут
- 500 товаров → 25–50 минут
- 1000 товаров → 50–100 минут

### Можно ли перенести заказы и клиентов?

Нет, скрипт работает только с товарами. Для переноса заказов используйте плагины Cart2Cart или LitExtension.

### Переносятся ли скидки?

Текущие цены переносятся как `Regular price`. Если в InSales есть `old_price` (зачёркнутая цена), она не переносится автоматически. Настройте скидки в WooCommerce вручную.

### Можно ли запустить миграцию повторно?

Да. При повторном запуске WooCommerce создаст новые товары (дубли). Чтобы обновить существующие — используйте опцию **Update existing products** при импорте (по SKU).

### Как перенести товары с вариантами?

Скрипт автоматически определяет вариантные товары. Если название варианта содержит `/` (например, «Красный / S»), значения разбиваются на атрибуты.

### Поддерживает ли скрипт русский текст?

Да. Все данные кодируются в UTF-8. Описания, названия, категории — всё переносится на русском.

### Где хранится API-ключ?

В файле `config.py`, который НЕ должен попасть в публичный репозиторий. Добавлен в `.gitignore`.

### Можно ли выгрузить только определённые категории?

В текущей версии — нет. Скрипт выгружает все товары в продаже. Фильтрация по категориям может быть добавлена в будущем.

### API InSales

Полная спецификация API: `openapi.json.insales.yaml` (в корне проекта).

Ключевые эндпоинты:
- `GET /products.json` — список товаров
- `GET /products/{id}/variants.json` — варианты товара
- `GET /products/{id}/images.json` — изображения товара
- `GET /categories.json` — категории
- `GET /accessories.json` — аксессуары (атрибуты)

Лимиты: 500 запросов в 5 минут.
